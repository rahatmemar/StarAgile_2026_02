Any application can be deployed in two ways;
	1. Manual deployment
	2. Automated deployment

Manual deployment of Java Application
-----------------------------------------------------------------------
Apache Tomcat

Clone the Code into the local ----> Generate the Artifact -----> Deploy the artifact in Tomcat (Manual way)

https://github.com/KastroVKiran/tomcat-app-kastro.git

Edit the context.xml file in manager and host-manager locations
<?xml version="1.0" encoding="UTF-8"?>
<Context privileged="true">
    <Valve className="org.apache.catalina.valves.RemoteAddrValve"
           allow="^.*$" />
</Context>

Edit the server.xml file with required port number

Edit the tomcat-users.xml file with admin info
  <role rolename="manager-gui"/>
  <role rolename="admin-gui"/>
  <user username="tomcat" password="tomcat" roles="manager-gui"/>
  <role rolename="manager-script"/>
  <user username="admin" password="admin" roles="manager-gui,admin-gui,manager-script"/>

JENKINS -
What is CI?
CI - Continuous Integration
Hotstar ----> Version 1 ----> Test the code ----> Build the code
Hotstar ----> Version 2 ----> Test the code ----> Build the code
Hotstar ----> Version 3 ----> Test the code ----> Build the code
....
CI = Continuous Test + Continuous Build

CD - Continuous Delivery
	if you are manually deploying the app into the servers then it is known as Continuous Delivery

CD - Continuous Deployment
	if you are automating the deployment of the app into the servers then it is known as Continuous Deployment

Jenkins Jobs;
Job is nothing but a task

Jenkins Jobs are of two types;
	1. Freestyle project		- every configuration related a job will be done using the Jenkins UI
	2. Pipeline 				- every configuration related a job will be done using scripts (Pipeline scripts)
	3. Multi-branch pipeline	- when the code is available in different branches of the same repo then we will use multi-branch pipelines

1. Freestyle project
In freestyle jobs we will have 6 options to configure;
	General
	SCM
	Triggers
	Environment
	Build Steps
	Post Build Actions
